# 网络诱饵

[English](cyber-decoys.md) | 中文

网络诱饵是本项目在网络空间中的分支：构造主机、服务、账号、文件、凭证、令牌或数据，使其只应被“错误的人”触碰。因此，任何交互本身都成为高置信告警。

实体诱饵消耗导弹，网络诱饵消耗攻击者的时间、注意力和技战术，并把原本静默的入侵转换为可观测事件。

## 为什么有效

- 正常用户不应访问蜜罐或使用蜜标。
- 入侵者在横向移动、凭证窃取和侦察阶段会主动触碰诱饵。
- 诱饵可以记录工具、命令、源地址、TTP 和攻击路径。
- 高交互诱饵能延迟攻击者并提高情报价值。

## 按交互深度

- **低交互**：模拟服务 banner 或简单协议响应。
- **中交互**：提供有限命令、文件系统或应用行为。
- **高交互**：更完整的主机、业务系统或工业控制环境。
- **分布式欺骗平台**：在真实环境中布设大量诱饵、假凭证和 breadcrumbs。

## 按诱饵对象

- 蜜罐主机。
- 蜜标文件和令牌。
- 假凭证。
- 假账号和假身份。
- 假数据库。
- 假 API / 服务。
- 假 Active Directory 对象。
- ICS/SCADA 协议仿真。

## 评估指标

- 告警精度。
- 攻击者交互深度。
- 发现横向移动的时间。
- 采集到的 TTP 数量。
- 对真实系统的隔离程度。
- 是否被攻击者指纹识别。

## 反诱饵

攻击者会通过默认配置、banner、时序、虚拟化痕迹、缺乏真实用户活动、文件系统单薄等方式识别蜜罐。网络诱饵的核心问题和实体诱饵一样：跨线索一致性。
